## Done

- **My Day home view** — unified timeline (default landing tab) merging a daily routine, today's to-dos, and work priorities. Routine builder (title, time, daily/weekday recurrence) with per-day completion; missed-and-undone routines drop off the home view and get reconciled in the Routine manager. Neon-backed (works on phone).
- **Work module** — team priority tracker: line items with P1–P4 priority, deadlines, team assignees (add-inline, dedup), labels, My Work / Team views, deadline agenda layout. Neon-backed (works on phone).
- **Finance module** — CSV/Excel statement import, per-issuer column mapping memory, category/subcategory tagging, local AI auto-categorization (Ollama), spend dashboard. Mac-local only (SQLite).
- **AI Assistant tab** — Open WebUI embedded in-app, voice mode (Kitten TTS + local Whisper), answers questions over live task/finance/gym data. Mac-local only.
- **Push notifications** — real Web Push for task reminders that fire when the app is closed (needs Vercel env vars + GitHub Actions secret set to go fully live).
- **Password manager** — zero-knowledge vault (🔒 header icon). Master password → client-side PBKDF2 + AES-GCM; server (Neon) only stores ciphertext, never the key or plaintext. Reveal/copy, strong-password generator. Works on phone.
- **Personal diary** — private journal (📖 header icon → full-screen glass overlay). One entry per calendar day, auto-saving as you write, optional mood emoji, ‹ › day navigation, searchable browse panel of past days. Neon-backed (works on phone). Future option: encrypt bodies with the vault crypto if desired.
- **File storage module** — Google-Drive-backed file manager (📁 header icon → full-screen glass overlay). Real `cosmos` folder in your own Drive via `drive.file` scope (cosmos only sees files it creates). Upload / download / create folders / rename / delete (→ Drive trash) / breadcrumb navigation / name search. Works on phone. Note: uses ~4.5MB Vercel request-body limit for uploads on the deployed side (no limit locally); large-file resumable upload is a future upgrade.
- **Finance: manage categories & subcategories** — new **Categories** tab in Finance. Add/rename/delete your own top-level categories and nest subcategories under them, each showing its transaction count. Deleting shows impact ("N subcategories go too, M transactions become uncategorized") and reassigns affected transactions to uncategorized (never orphans a reference). Reflects everywhere categories are used. Mac-local.
- **Finance: tagging feedback, manual correction & labels** — tap any transaction (not just untagged) to change its category, with **"re-tag all past <merchant> too"** for bulk corrections and a **Clear** to un-categorize. Corrections upsert a merchant rule AND feed in as **few-shot examples** to the local-LLM categorizer so it learns your preferences. Freeform **labels** (e.g. "reimbursable", "work trip") with suggestions from past labels, shown as chips on rows and filterable in Transactions. Plus a per-transaction **notes** field. Mac-local.
- **Finance: dashboard + filters redesign** — Overview reworked with the dashboard-designer skill: hero "spent this month" KPI with month-over-month delta, context KPIs, ranked colored category breakdown with %, Indian number formatting (₹9.3L), glass surfaces. Transactions filters rebuilt: type (spend/income), categorization **status** (all/categorized/uncategorized), category dropdown, label chips, amount range, quick date presets + custom range, active-filter chips, and a result summary (count · out · in). The "needs tagging" banner counts **this month only**, not the all-time backlog.
- **Finance: direction-aware categorizer** — the local-LLM categorizer now knows money-in vs money-out. Prompt tags each txn `[money IN]`/`[money OUT]`; credits can only be income-side categories (Income/Refunds/Cashback/Transfers), never a spend category. Rules + grouping + apply are keyed by (merchant, direction), so a refund never inherits a merchant's spend category. Fixed 37 existing mis-tagged credits. Mac-local.
- **Finance: categorization progress + control** — the AI categorize run persists to the DB, so progress **survives a refresh** (re-adopts and polls a running job); it keeps running even if the tab closes; a **Stop** button cancels gracefully (aborts the in-flight model call, keeps tagged work, offers Resume). Mac-local.
- **Finance: source framework + multi-file import** — per-source processors (SBI, Axis ACE) with password-protected file decryption + strong dedup (txn_uid). Categories tab locks the seeded default taxonomy (only your own categories are editable). **Multi-file import**: select many statements at once, preview the first, import all sequentially with per-file results and automatic overlap de-duplication. Mac-local.
- **Finance Assistant (local text-to-SQL)** — ask money questions in the Assistant on a local model (Qwen2.5 72B added to the switcher): local qwen writes a read-only SQL query from the schema + your question, it runs on a read-only connection to the local finance DB, and the local model answers from the rows. The finance tool is only offered on **local** models, so finance never reaches the cloud. Mac-local.
- **Overlay navigation — dismiss from anywhere** — every full-screen overlay (Vault, Files, Diary, Assistant) now has a top **grabber** you can tap or swipe down to close, plus **Esc** to dismiss. Shared helper so all four stay consistent.
- **Assistant: markdown in chat** — assistant replies render as real markdown (headings, bold/italic, bullet + numbered lists, inline + fenced code, links, blockquotes) via a lightweight dependency-free renderer styled with the glass tokens. User messages stay plain.
- **To-do: sub-items (subtasks)** — add sub-items under any to-do (inline "+ sub-item"); the parent shows **progress (2/3)** and **auto-completes when every sub-item is checked** — and re-opens if you uncheck one or add a new open sub-item. Checking a parent checks all its sub-items; deleting a parent removes them. One level of nesting, `parent_id` on tasks. Neon-backed (works on phone).
- **Pomodoro / focus timer** — a focus timer at the top of the Work module: progress ring + MM:SS, configurable focus/break/long lengths (default 25/5/15, long break every 4), start/pause/resume/reset/skip, cycle dots, and an optional "focus on" picker of your open work items. Timestamp-based (remaining derived from a stored end-time in localStorage) so it survives refresh / tab-switch / phone-lock; two-tone chime + a local notification on phase end; focus auto-rolls into a break. Works on phone. (Future: server-scheduled push so it also alerts when the phone is fully locked.)
- **Work: drag-reorder P1s** — the P1 "🔴 Now" strip is now manually orderable: each P1 row has a ⠿ grip you drag (touch + mouse) to reorder, persisted via a `sort_order` column and `/api/work/reorder`. P1s sort by that manual order (created-at as tiebreak). Neon-backed (works on phone). Non-P1 items stay deadline-bucketed.
- **Assistant split + chat history** — the assistant is now two contexts. The **general assistant** (header ✨) has no finance access and **persists chat history to Neon** (thread list, resume, rename, delete, new-chat) — safe because it never touches finance. The **finance assistant** is embedded on the Finance Overview ("Ask about your finances"): local-model only, has the `query_finance` tool, and is **ephemeral** (nothing stored), so finance data never lands anywhere. Enforced server-side: the `finance` flag + a local model gate the finance tool.
- **Medical history** — a health module (🩺 header icon → full-screen overlay) with **separate profiles per person** (Mom / Dad / Self / …, each with relation, blood group, DOB). Per-profile **timeline** of records — **Visit / Test / Prescription / Note** — each with date, doctor, clinic, and details. **Attach scans + PDFs** which upload to a "Medical" folder in your Google Drive (`drive.file`); the app stores only refs and streams them back inline to view. Add/edit/delete records + profiles. Neon-backed metadata + Drive files (works on phone). Future: search across records, follow-up reminders, structured medicines. Tables: health_profiles / health_records / health_files.
- **Finance: manual cash entries** — an **+ Add** button in Transactions opens a manual-entry sheet (Spent/Received, amount, what-for, date, account, category) for cash spends and other transactions that never hit a statement. Defaults to an auto-created **Cash** account; each entry gets a unique id so it never collides with imports. Mac-local.
- **Finance Overview: donut + heatmap + transfers split** — "Where it goes" is now an SVG **donut** (center = categorized spend, top-5 + Other, ranked legend) plus a **Top merchants** list, and a **daily-spend calendar heatmap** (IST layout, p90+sqrt scaling so outlier days don't wash out the rest, biggest-day/no-spend-days callout). **Transfers** are excluded from all spend figures (hero, donut, heatmap, merchants) and surfaced as their own "moved, not spent" tile; Fees & Charges are kept as spend. Mac-local.
- **Phone access groundwork** — `trustHost` so cosmos accepts the Tailscale HTTPS hostname (auto host detection); enables using the Mac-local features (Finance, local-qwen Assistant) from the phone over a Tailscale tunnel.
